
## 2025-02-12 - [Performance bottleneck in property accesses with primitive types]
**Learning:** `JSONDecoder().decode(...)` and `JSONEncoder().encode(...)` introduce significant overhead when serializing/deserializing arrays of primitive types like `[Double]` or `[String]`. This is extremely problematic in properties like `embeddingVector` in `Memory.swift` because these values are accessed on every single item during `O(n)` linear loops, such as semantic searches via `vDSP_dotprD`. The wrapper structs generated by Swift's native Codable logic slow down the operations far beyond raw vector calculations.
**Action:** Always prefer using `JSONSerialization.jsonObject(with:) as? [Type]` and `JSONSerialization.data(withJSONObject:)` for primitive structs and arrays within high-frequency getters or initialization paths.
